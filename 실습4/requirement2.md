# 실습 4 요구사항

## 1. 실습 개요

- 주제: 시각화 4종 · 통계 검정 · sklearn Pipeline
- 입력 데이터: `sales_100k.csv`
- 실습 3의 산출물을 실습 4에서 연계하여 사용한다.
- 작업 목적: 실습 3에서 정제·집계한 데이터를 시각화, 통계 검정, 머신러닝 Pipeline으로 확장한다.
- 제출 파일명: `캠퍼스명_반_이름.py`
- 총점: 100점

## 2. 실습 3 → 실습 4 연계

| 실습 3 산출물 | 실습 4 활용 방법 |
| --- | --- |
| IQR 이상치가 제거된 DataFrame | 시각화 및 통계 검정의 입력 데이터로 사용 |
| `region-category` groupby 결과 | 지역·카테고리 분석 및 카이제곱 검정에 활용 |
| `sales_100k.csv` | Pipeline의 학습 데이터 원본으로 사용 |

### 연계 작업 흐름

1. `sales_100k.csv`를 불러온다.
2. 실습 3과 동일하게 결측치 및 IQR 이상치를 처리한다.
3. 정제된 DataFrame으로 EDA 시각화와 통계 검정을 수행한다.
4. 원본 또는 목적에 맞게 변형한 데이터로 sklearn Pipeline을 학습·평가한다.
5. 학습 모델과 Plotly 차트를 파일로 저장한다.

## 3. 필수 구현 요구사항

### 3.1 EDA 시각화 4종

- [ ] `fig, axes = plt.subplots(2, 2)`를 사용하여 하나의 Figure에 2×2 서브플롯을 구성한다.
- [ ] 다음 차트 4종 또는 이와 유사한 조합을 작성한다.
  - 히스토그램 + KDE
  - 박스플롯
  - 월별 라인 차트
  - 상관 히트맵
- [ ] 4개 차트를 각각 별도의 `plt.show()`로 출력하지 않는다.
- [ ] 제목, 축 이름, 범례 등 차트에 필요한 정보를 표시한다.
- [ ] 한글이 깨지지 않도록 실행 환경에 맞는 한글 폰트를 설정한다.
  - macOS 예시: `AppleGothic`
  - Windows 예시: `Malgun Gothic`

> 강의 음성에서는 히스토그램·박스플롯·막대그래프·선형그래프도 언급되었지만, 제시된 실습 화면의 명시적 조합은 히스토그램+KDE·박스플롯·월별 라인·상관 히트맵이다. 제출 시에는 화면에 명시된 조합을 우선 적용한다.

### 3.2 통계 검정

#### 독립표본 t-test

- [ ] 비교할 두 그룹을 선택한다. 기본 예: 서울과 부산의 평균 매출 비교
- [ ] 성별이나 다른 두 지역처럼 분석 의도에 맞는 두 그룹을 선택해도 된다.
- [ ] `scipy.stats.ttest_ind`를 적용한다.
- [ ] 통계량과 p-value를 출력한다.
- [ ] `p < 0.05`를 기준으로 통계적 유의성 여부를 판단한다.
- [ ] 검정 결과의 의미를 한 줄로 출력한다.

#### 카이제곱 독립성 검정

- [ ] 두 범주형 변수를 선택한다.
  - 강의 기본 방향: 지역(`region`)과 카테고리(`category`)
  - 선택 가능 예시: `category`와 `payment_method`
- [ ] 두 변수의 분할표를 생성한다.
- [ ] 분할표에 `chi2_contingency`를 적용한다.
- [ ] p-value를 출력한다.
- [ ] `p < 0.05`를 기준으로 두 변수의 독립성 여부를 판단한다.
- [ ] 검정 결과의 의미를 출력한다.

### 3.3 sklearn Pipeline 구성 및 저장

- [ ] `ColumnTransformer`와 `Pipeline`을 사용한다.
- [ ] 전처리 단계와 모델 1개 이상을 하나의 Pipeline 객체로 구성한다.
- [ ] 예측 모델은 선형 모델 등 학습자가 원하는 모델을 자유롭게 선택할 수 있다.
- [ ] 모델 1개만 사용해도 되며, 원하는 경우 3~4개 모델을 비교·검증해도 된다.
- [ ] 실습의 핵심은 특정 모델의 성능 경쟁이 아니라 전처리와 모델을 Pipeline으로 연결하는 것이다.
- [ ] Pipeline을 사용하여 다음 작업을 순서대로 수행한다.
  1. 훈련(`fit`)
  2. 예측(`predict`)
  3. 평가(`score`)
  4. 저장
  5. 재로딩
- [ ] `joblib.dump()`를 사용하여 `.pkl` 또는 `.joblib` 모델 파일을 생성한다.
- [ ] 저장한 모델 파일을 다시 불러온다.
- [ ] 필요하다면 분석 목적에 맞게 데이터나 변수 구성을 변형할 수 있다.
- [ ] 코드 또는 데이터 오류가 발생하면 의도에 맞게 수정하여 실행 가능한 상태로 완성한다.

### 3.4 Plotly 인터랙티브 차트

- [ ] 지역·카테고리별 총매출을 집계한다.

## 8. 구현 시 허용되는 선택

- 모델 종류는 자유롭게 선택할 수 있다.
- 모델은 1개 이상 사용하며, 여러 모델을 비교해도 된다.
- 분석 목적에 맞게 데이터를 변형하거나 변수를 선택할 수 있다.
- t-test의 두 그룹은 서울·부산을 기본 예시로 하되 다른 성별·지역 그룹을 선택할 수 있다.

- 단, 선택한 데이터·변수·모델의 의도는 주석이나 출력문으로 명확히 설명한다.
